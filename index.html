Documentação - XSalle

SOBRE

O XSalle é uma aplicação web, desenvolvida na ferramenta Buuble, que tem como objetivo facilitar negociações, oportunidades, compras e vendas no portal da Petronect.

O sistema é integrado a um Bot (Robô de Software) que roda em uma máquina virtual e realiza a busca de oportunidades de compras e vendas no portal da Petronect, e envia para o aplicativo web, onde o usuário pode visualizar as oportunidades e realizar operações especificas de maneira mais eficiente, rápida e segura.

O sistema trabalha com diversas APIs que são executadas através de workflows, que são executados através de eventos no Bubble. Também possuí conexão com banco de dados mySQL armazenado no AWS (Amazon Web Services), que é onde o robô envia e registra as informações das oportunidades e o XSalle consome essas informações através de APIs.

BANCO DE DADOS

O XSalle possuí um banco de dados mySQL armazenado no AWS (Amazon Web Services), que é onde o robô envia e registra as informações. O XSalle consome via API as informações do banco de dados para gerenciar as oportunidades, cotações e salas de colaboração.

O banco de dados é conectado via API com o Bubble através de um plugin chamado SQL Database Connector.

API's

O XSalle trabalha com diversas APIs que são executadas através de workflows, as API's são responsáveis pelas requisições e distribuição de informações por todo o sistema. As API's são configuradas através de um plugin chamado API Connector.

OPORTUNIDADES

A página de oportunidades é a página principal do XSalle, onde o usuário pode visualizar as oportunidades de compras e vendas que foram enviadas pelo Bot. A página é no estilo kaban e as oportunidades são separadas por padrão em 3 colunas: NOVAS OPORTUNIDADES, DECLINADAS e ACEITAS. O usuário também pode adicionar nova colunas para melhor organização e mover as oportunidades entre as colunas.



Print da página Oportunidades


Cada coluna tem uma comunicação via API com o Bot, onde o usuário pode realizar operações de aceitar ou declinar as oportunidades.


NOVAS OPORTUNIDADES - Nessa coluna são apresentadas as oportunidades que o Bot encontrou no portal da Petronect e enviou para o XSalle.


DECLINADAS - Nessa coluna são apresentadas as oportunidades que o usuário declinou.


ACEITAS - Nessa coluna são apresentadas as oportunidades que o usuário aceitou.



As informações preenchidas nos cards das oportunidades são enviadas pelo robô através de uma API para o banco de dados armazenado no AWS que armazena as informações como "data types" com nome de "get_listas oportunidade" (que também é o nome da API).

Parent group's API get_listas oportunidade tem os seguintes dados armazenados pela API em data fields:

  • 's id_oportunidade
  • 's id_lista
  • 's nome
  • 's id_petronect
  • 's criterio_julgamento
  • 's data_fim_periodo_cotacao
  • 's moeda
  • 's local_entrega
  • 's tipo
  • 's empresa
  • 's cor
  • 's ativo
  • 's visualizada
  • 's comentarios
  • 's replicado_petronect
  • 's raw body text

Para exibição do card da oportunidade da tabela, o sistema faz o seguinte Query:


Data Type - Parent group's API get_listas oportunidade;
Field - 's id_petronect e 's nome;


Ao mover o card da oportunidade para outra coluna, o sistema faz o seguinte Workflow:
Step 1 - Exibe um loading na tela;
Step 2 - Faz uma requisição a "API Get Listas" para executar a ação da coluna que o usuário moveu o card e mover o card no banco de dados;
Step 3 - Recarrega o elemento das oportunidades "board_fapi_oportunidades";
Step 4 - Exibe as oportunidades filtradas de acordo com a coluna que o usuário moveu o card, para isso faz uma quesição a "API Get Listas" para gerar a lista após a execução anterior;
Step 5 - Esconde o loading da tela;
